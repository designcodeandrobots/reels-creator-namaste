# Playbook: анализ блогера → скил для рилсов

> Инструкция для ИИ-агента (чат-бота). Работай **строго по шагам**, не перепрыгивай. После каждого шага — короткая контрольная точка: подтверди результат с пользователем, прежде чем идти дальше. Если на шаге чего-то не хватает (данных, файлов, доступа) — остановись и попроси.

## Вступление, которое агент говорит пользователю

«Я помогу тебе проанализировать рилсы любого блогера. Сначала мы скачаем данные за последние 100 рилсов, скачаем аудио, переведём в текст и сделаем анализ. После создам скил на поиск тем и написание сценариев. Дальше можно будет создать контент-план.»

Затем спроси: **ник блогера** в Instagram и **нишу** (маркетинг, дизайн, фитнес и т.д.), и идём по шагам.

---

## Шаг 0. Вход
Собери от пользователя:
- `@ник` блогера;
- нишу/тематику (для будущего анализа и скила);
- ОС пользователя (Mac / Windows) — влияет на выбор инструмента скачивания аудио;
- куда складывать рабочие файлы (папка проекта).

Создай в папке проекта структуру: `data/` (таблицы), `audio/` (аудио), `transcripts/` (тексты), `analysis/` (анализ), `skill/` (итоговый скил), и файл `progress.md` для логирования прогресса.

**Контрольная точка:** структура папок создана, ник и ниша зафиксированы.

---

## Шаг 1. Получение данных по блогеру (парсер)
Инструмент: **https://github.com/designcodeandrobots/insta-reals-parser**

1. Клонируй и установи по инструкции из README репозитория парсера.
2. Запусти парсер на `@ник`, ограничь выгрузку **последними 100 рилсами**.
3. На выходе — **таблица** (CSV/XLSX) с данными по роликам. Главное, что нам нужно: **URL каждого рилса** (и желательно просмотры/дата/описание — пригодится для ранжирования).
4. Сохрани таблицу в `data/`.

**Контрольная точка:** в `data/` лежит таблица со ~100 строками и колонкой URL. Покажи пользователю количество строк и первые 5 URL.

---

## Шаг 2. Скачивание аудио по URL из таблицы
Нужно скачать аудиодорожку каждого ролика из колонки URL.

**Mac (платно, удобно):** Downie — кидаешь ссылки, выбираешь «извлечь аудио».

**Бесплатные альтернативы (Mac и Windows):**
- **yt-dlp** (рекомендую, кросс-платформенный, CLI) + **ffmpeg** для извлечения аудио. Пример: `yt-dlp -x --audio-format mp3 -a urls.txt` (где `urls.txt` — список URL из таблицы). Ставится: Mac — `brew install yt-dlp ffmpeg`; Windows — `winget install yt-dlp ffmpeg` или `choco install yt-dlp ffmpeg`.
- **JDownloader 2** — бесплатный GUI, кросс-платформенный, принимает пачку ссылок, умеет извлекать аудио.
- **cobalt.tools** — бесплатный веб-сервис, без установки (для небольших пачек, вручную).
- **4K Video Downloader / 4K Tokkit** — бесплатный тариф, GUI, Mac/Windows.
- **instaloader** (CLI, Python) — заточен под Instagram, тянет рилсы; аудио потом извлечь через ffmpeg.

Алгоритм для агента:
1. Выгрузи все URL из таблицы в `data/urls.txt` (по одной ссылке на строку).
2. Предложи пользователю инструмент под его ОС (по умолчанию — yt-dlp как бесплатный кросс-платформенный).
3. Дай готовую команду/инструкцию. Скачанные файлы — в `audio/`.

**Контрольная точка:** в `audio/` лежит ~100 аудиофайлов. Сверь количество с числом строк в таблице, сообщи о пропусках.

---

## Шаг 3. Транскрипция аудио → текст
Инструмент: **https://github.com/designcodeandrobots/voice-to-text** — бесплатно, через **локальную LLM** (без облака и оплаты).

1. Клонируй и установи voice-to-text по его README (модель, зависимости).
2. Прогони все файлы из `audio/` через транскрипцию.
3. Каждую расшифровку сохрани отдельным `.txt` в `transcripts/`. Имя файла = осмысленное название ролика или его ID/описание из таблицы.

**Контрольная точка:** в `transcripts/` лежит ~100 `.txt`. Открой 2–3 для проверки качества расшифровки (нет ли пустых/обрезанных). Сообщи о браке.

---

## Шаг 4. Анализ контента
Когда все тексты собраны — проведи анализ контента блогера.

Разбери по транскрипциям и зафиксируй:
- **Темы и рубрики** — о чём блогер снимает, как группируются темы.
- **Что делает контент цепляющим** — приёмы, тон, «фишка» автора.
- **Структурные паттерны** — типовые каркасы роликов.
- **Анатомия хука** — как сделаны первые 2–3 секунды; собери топ реальных хуков.
- **Мост** — как переходит от хука к своей теме.
- **CTA** — механика призыва (кодовые слова, подпись, ссылки).
- **Тон и стиль** — лексика, фразовые маркеры, обращение к зрителю.
- **Аудитория**.
- **Выводы для генератора сценариев**.

Сохрани результат в `analysis/analysis.md`. По желанию — выгрузи в Notion отдельной страницей.

**Контрольная точка:** `analysis/analysis.md` создан и покрывает все пункты выше. Покажи пользователю оглавление и «главное за 30 секунд».

---

## Шаг 5. Создание скила (поиск тем + написание сценариев)
На основе анализа собери скил в `skill/SKILL.md`. Скил должен уметь **два режима**:
- **Генерация тем** — пачка ранжированных идей (хук + мост + потенциал), с проверкой уникальности по стоп-листу.
- **Полный сценарий** — по структуре: хук → развитие → мост → инсайт (с разворотом на зрителя) → CTA + подпись.

Обязательные элементы скила:
- описание (`description`) с триггерами вызова;
- правила «дерзости» = честное + неудобное + конкретное (или другой стержень стиля автора);
- карта ниши (чтобы темы были широкими, а не однотипными);
- словарь фразовых маркеров автора;
- 1–2 **эталонных разобранных сценария** (критично для холодного старта на слабых моделях);
- файл `skill/covered-topics.md` — стоп-лист снятых тем для контроля уникальности.

**Контрольная точка:** `skill/SKILL.md` готов, проверь его — сгенерируй 10 тестовых тем и 1 сценарий, оцени, попадает ли в стиль. При провале — доработай скил.

---

## Шаг 6. Дальше: контент-план
Когда скил работает — предложи собрать **контент-план**: N тем на период, распределённых по рубрикам и типам хука, с проверкой по стоп-листу, готовых к раскрытию в сценарии.

---

## Принципы для агента
- Иди по шагам, не перепрыгивай; на каждом — контрольная точка.
- Логируй прогресс в `progress.md` (что сделано, что дальше) — чтобы можно было продолжить в новом чате.
- Все промежуточные артефакты сохраняй в файлы, не держи только в чате.
- Не выдумывай данные: нет таблицы/аудио/текста — остановись и попроси.
- Уважай авторские права и правила площадок: материалы используются для анализа стиля, а не для копирования контента 1:1.
