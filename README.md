# reels-analyzer — пайплайн анализа блогера и создания скила

Привет 👋 Я помогу тебе **проанализировать рилсы любого блогера** и собрать на их основе рабочий инструмент.

Маршрут такой:
1. Скачаем данные по последним **100 рилсам** блогера.
2. Скачаем аудио из этих роликов.
3. Переведём аудио в текст (транскрипция).
4. Сделаем **анализ** контента и сохраним его в файл.
5. Соберём **скил**, который умеет искать темы и писать сценарии в стиле этого блогера.
6. Дальше на этом можно строить **контент-план**.

Это репозиторий-инструкция: основной файл — [`playbook.md`](./playbook.md). Он написан так, чтобы по нему мог пройти **чат-бот / ИИ-агент по шагам**, останавливаясь на контрольных точках.

## Используемые инструменты
- **Парсер данных:** [insta-reals-parser](https://github.com/designcodeandrobots/insta-reals-parser) — выгружает таблицу по рилсам блогера (включая URL роликов).
- **Скачивание аудио:** Downie (Mac, платно) или бесплатные альтернативы — yt-dlp, JDownloader2, cobalt.tools (см. playbook, шаг 2).
- **Транскрипция:** [voice-to-text](https://github.com/designcodeandrobots/voice-to-text) — бесплатно, через локальную LLM (без облака и оплаты).
- **Анализ + скил:** ИИ-агент (этот чат) по инструкции из `playbook.md`.

## Запуск для AI-чатов (ChatGPT, Claude, Gemini и др.)

Инструкция рассчитана на ИИ-агента с доступом к файлам/командам (Cowork, Claude Code, агент с тулзами). В обычном чат-боте без инструментов шаги 1–3 (установка парсера, скачивание аудио, транскрипция) пользователь выполняет сам, а бот ведёт по процессу и делает шаги 4–5 (анализ и скил).

**Шаг 1. Дай боту контекст.** Скопируй в чат содержимое `playbook.md` (или дай ссылку на этот репозиторий, если у бота есть веб-доступ).

**Шаг 2. Вставь стартовый промпт:**

```
Ты — ассистент по анализу рилсов. Работай строго по playbook.md, шаг за шагом,
с контрольной точкой после каждого шага — не перепрыгивай.

Блогер: @НИК
Ниша: НИША
Моя ОС: Mac / Windows
Папка проекта: ПУТЬ

Начни с Шага 0: задай уточняющие вопросы, создай структуру папок и progress.md,
затем веди меня по шагам. На каждом шаге показывай результат и жди моего «ок»,
прежде чем идти дальше. Не выдумывай данные — если чего-то не хватает, остановись
и попроси.
```

**Шаг 3. Идите по шагам.** Бот спросит недостающее, выполнит/подскажет инструменты, сохранит артефакты в файлы и будет логировать прогресс в `progress.md`. Если процесс прервался — в новом чате снова дай `playbook.md` + `progress.md` и скажи «продолжай с последнего незавершённого шага».

**Совет:** если у бота нет доступа к терминалу, на шагах 1–3 проси «дай мне точные команды для моей ОС» — выполнишь их сам и вернёшь боту результат (таблицу / файлы / тексты).

## Быстрый старт
Открой `playbook.md` и отдай его чат-боту целиком (или скажи: «работай по playbook.md, шаг за шагом»). Бот проведёт тебя от установки парсера до готового скила.
